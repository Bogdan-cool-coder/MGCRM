# MACRO Global CRM — правила для Claude Code

> **Дизайн-система — это эталон (единственный источник истины) для всего визуала.**
> Она лежит в `.claude/skills/macroglobal-design/`. Перед созданием или изменением
> ЛЮБОГО UI (страница / модуль / компонент / кнопка / иконка / отступ / цвет) ты
> ОБЯЗАН сверяться с ней. Не сочиняй визуал «на глаз».

## Что читать перед UI-работой (всегда)
1. `.claude/skills/macroglobal-design/README.md` — Visual / Content foundations, иконография, и **профессиональная критика страниц Сделки/Контакты/Задачи** (это наш agenda на модернизацию).
2. `.claude/skills/macroglobal-design/tokens/*.css` — каноничные значения (`--mg-*`).
3. `.claude/skills/macroglobal-design/components/` и `ui_kits/crm/` — эталонные компоненты и экраны.

## Как это ложится на код репозитория
В рантайме код использует **существующие токены репо**, которые реализуют этот эталон
(значения совпадают — система выведена из репо):
- **Цвет / поверхности / границы:** SCSS-переменные `front/src/theme/scss/foundation/_colors.scss` (`$primary-900`, `$surface-0…900`, `$status-*`, `$color-danger`…) и `--p-*` от PrimeVue. Первоисточник значений — `front/src/theme/tokens/colors.ts`.
- **Типографика:** `$font-size-*`, `$font-weight-*`, `$line-height-*` (`_typography.scss`). База интерфейса — **14px**.
- **Отступы:** `$space-1…8` (4→32, шаг 4px). Гаттер страницы — 24px, паддинг карточки — 16px.
- **Радиусы:** `$radius-sm|md|lg|xl` (4/6/8/12). Дефолт кнопки/карточки — `md` (6px).
- **Тени:** `$shadow-sm|card|md|lg`. Карточка = бордер 1px + `$shadow-card`.
- **Тема PrimeVue:** меняется ТОЛЬКО через пресет `front/src/theme/adapters/primevue/`, не точечными оверрайдами.

**Правило сопоставления:** значение из дизайн-системы (`--mg-*`) ⇄ соответствующая
переменная репо (`$…` / `--p-*`). В `.vue`/`.scss` пиши переменную репо, НЕ литерал.

## Жёсткие правила (нарушение = переделать)
- ❌ Никаких хардкод-цветов (`#xxxxxx`, `rgb(...)`) в `.vue`/`.scss`, кроме объявления самих токенов и задокументированных бренд-инвариантов (сайдбар `#172747`, шапка карточки сделки `#172747`).
- ❌ Никаких «магических» px-отступов вне шкалы `$space-*`; радиусов вне `$radius-*`.
- ❌ Никаких новых цветов, градиентов, bluish-purple, цветных теней/glow, эмодзи.
- ❌ Не рисовать иконки вручную SVG. Иконки — **PrimeIcons** (`pi pi-*`); бренд-гайд допускает Bootstrap Icons для маркетинга.
- ✅ Бренд — навигационный нэйви `#172747` (+ `#0E172B` / `#2B4987`), нейтральная серая шкала, статусы мягкими тинт-пилюлями.
- ✅ Шрифт — Inter (веб-фолбэк SF UI Display), Roboto для документов.
- ✅ Состояния: hover — на шаг светлее/темнее; focus — нэйви-бордер + 2px ring; без bounce/scale.

## Копирайтинг (RU-first)
- Русский, нейтрально-деловой, вежливое «вы». Кнопки — глаголы в sentence-case: «Создать сделку», «Применить», «Сбросить». Без эмодзи.
- Деньги: символ ПОСЛЕ суммы через пробел, пробелы в разрядах — `1 200 000 ₽`, `≈ 6,8 млн ₽`. Даты `ДД.ММ.ГГ`.

## Проверка перед коммитом
Прогоняй adherence-линт `npm run lint:ds` (из `front/`) — это **stylelint** с запретом
хардкод-значений мимо токенов, по двум раздельным конфигам: `front/.stylelintrc.vue.json`
для `.vue` и `front/.stylelintrc.scss.json` для `.scss` (общая база — `front/.stylelintrc.json`).
В репо тот же `lint:ds` запускается как **pre-commit** через `.githooks/pre-commit`
(`git config core.hooksPath .githooks`; это НЕ husky). Падает на твоём значении —
заменяй на токен, не глуши правило. Для React/TSX-поверхностей есть отдельный
`_adherence.oxlintrc.json` (валидирует пропсы компонентов и ловит hex/px).
