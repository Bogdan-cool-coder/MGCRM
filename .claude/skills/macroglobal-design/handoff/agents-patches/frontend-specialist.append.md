## 🅐 Дизайн-система MACRO Global — ЭТАЛОН токенов и компонентов

Источник истины по визуалу — **`.claude/skills/macroglobal-design/`** (перебивает
vault-спеку и Vizion-визуал). Перед кодом сверься с `README.md` + `tokens/*.css` и
реализуй ТЗ `designer` 1-в-1.

**Сопоставление токенов системы ↔ переменные репо** (значения совпадают — система выведена
из репо; в `.vue`/`.scss` пиши переменную репо, НЕ литерал и НЕ `--mg-*` напрямую):
- цвет/поверхности/границы → `$primary-900`, `$surface-0…900`, `$status-*`, `$color-danger…`
  (`theme/scss/foundation/_colors.scss`) и `--p-*`;
- типографика → `$font-size-*`, `$font-weight-*`, `$line-height-*` (база интерфейса 14px);
- отступы → `$space-1…8` (4→32); радиусы → `$radius-sm|md|lg|xl`; тени → `$shadow-*`.
Тему PrimeVue меняешь ТОЛЬКО через пресет `theme/adapters/primevue/`, не точечными оверрайдами.

## 🅑 Reuse-first (строго)
Сначала ищи готовый компонент в `front/src/components/**`, базовые PrimeVue и эталонные
компоненты системы. **Новый компонент — только если `designer` обосновал в ТЗ.** Не плоди
дубли (Button/Tag/Avatar/Card/KanbanCard уже есть как эталон).

## 🅒 Обе темы — обязательно
Каждая новая/изменённая поверхность работает в **светлой И тёмной** теме. Используй
семантические токены (`$surface-card`, `--p-*`), помни про инвертированную dark-палитру
(см. `theme/adapters/primevue/semantic/foundation.ts`). Никакого хардкода цвета, который
ломается в одной из тем. Перед остановкой — проверь обе.

## 🅓 Adherence-линт перед остановкой
Добавь к чек-листу остановки: `npm run lint:ds` (stylelint, запрет hex/px мимо токенов) —
без ошибок. Падает на твоём значении → заменяй на токен, не глуши правило.
