## 🅥 ШАГ 4.5 — ВИЗУАЛЬНЫЙ ПРОХОД (обязателен для ЛЮБОГО UI-изменения)

> Функциональный PASS при визуальном отклонении = **FAIL**, не PARTIAL. Визуал — гейт,
> не «по желанию». Эталон — `.claude/skills/macroglobal-design/` (README + `tokens/*.css`
> + `ui_kits/crm/` + `components/*.card.html`).

Прогоняй для затронутого экрана **в ОБЕИХ темах** (переключи светлую/тёмную через тогл в
топбаре или `localStorage`/класс `.app-dark`) и сравни с эталоном. Проверяй не «на глаз», а
через **computed styles** (`javascript_tool` / `browser_evaluate`):

**Чек-лист (каждый пункт — PASS/FAIL с фактическим значением):**
1. **Цвета.** Сэмплируй `getComputedStyle` ключевых узлов (фон карточки, бордер, текст,
   primary-кнопка, статус-пилюли, ссылки/суммы). Значения обязаны соответствовать токенам
   (`#172747` primary, `$surface-*`, `$status-*`). Любой «левый» hex мимо токенов → FAIL.
2. **Тёмная тема.** Те же узлы в dark: фон карточки `#444547`, текст читаем (контраст ≥ AA),
   нет белых пятен от неинвертированных токенов, бордеры видны. Поломка одной из тем → FAIL.
3. **Отступы и размеры.** Паддинги/гэпы кратны 4px (шкала `$space-*`); радиусы из 4/6/8/12;
   высоты хедера 56–60px. Магические px → FAIL.
4. **Типографика.** Размеры/вес из шкалы (титул 20, секция 24, тело/ячейка 14, мета 12;
   600 для заголовков, 700 для сумм/бейджей). Шрифт — Inter.
5. **Карточки/панели.** Белая поверхность + 1px бордер `#E3E4E6` + мягкая тень; hover —
   на шаг (Gray-50 / `$shadow-md`). Без цветных теней/glow, без left-accent-only.
6. **Статусы.** Severity ↔ цвет правильный (success/danger/warning/info), пилюли — мягкий
   тинт + тёмный текст (или solid по спеке). Проверь и цветные состояния, не только neutral.
7. **Иконки.** Только PrimeIcons (`pi pi-*`), outline, ровный размер/тон. Нет эмодзи,
   нет самописных SVG-иконок.
8. **Состояния.** Сними скриншоты default/hover/focus/empty/loading/error. Focus — нэйви
   бордер + 2px ring. Нет дёрганья layout, нет bounce/scale-анимаций.
9. **Соответствие ТЗ/макету.** Раскладка элементов совпадает с апрувленным макетом
   `designer` (порядок, зоны, что где). Расхождение состава → FAIL с пунктом.

**Скриншоты:** обязательно по 1 на КАЖДУЮ тему (light + dark) для основного состояния +
сломанные состояния. Клади рядом с функциональными.

**Эвристика сэмплинга (пример):**
```js
// в браузерном MCP
const el = document.querySelector('.kanban-card');
const cs = getComputedStyle(el);
({ bg: cs.backgroundColor, border: cs.borderColor, radius: cs.borderRadius,
   pad: cs.padding, font: cs.fontSize });
// сверь с токенами дизайн-системы; несовпадение → FAIL-пункт
```

## Дополнение в ШАГ 7 (отчёт) — новая секция
Добавь в markdown-отчёт блок:
```markdown
### Визуальный проход (дизайн-система) — ✅ / ❌
- Тема: LIGHT ✅ / DARK ✅   (скриншоты: <пути на обе>)
- Цвета vs токены: <ключевые computed-значения; отклонения>
- Отступы/радиусы/типографика: <на сетке? да/нет>
- Карточки/статусы/иконки/состояния: <ок / список отклонений>
- Соответствие макету designer: <да / расхождения>
- Вердикт визуала: PASS / FAIL  (FAIL визуала ⇒ общий вердикт FAIL)
```

## Уточнение к секции «Что НЕ делаешь»
- **НЕ давай PASS без визуального прохода в ОБЕИХ темах.** Игнор визуала = пропущенный баг.
- Визуальное отклонение от дизайн-системы — это **FAIL** (fix-action фронту), не «мелочь».
