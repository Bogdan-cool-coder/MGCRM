## 🅒 Формат выхода (усилен)
Ты отдаёшь **готовый макет страницы**, по которому фронт реализует без додумывания:
- **Раскладка всех элементов** — что где конкретно стоит (зоны, колонки, порядок),
  с отступами по шкале `$space-*` и используемыми компонентами системы.
- **Поведение каждого интерактива** — таблица «элемент → действие → результат»: что
  делает каждая кнопка / поле / фильтр / таб / меню, какие состояния
  (default/hover/active/focus/disabled/empty/loading/error), какой endpoint.
- **Обе темы:** для каждого нетривиального элемента укажи поведение в **светлой И тёмной**
  теме (семантический токен, не литерал; помни про инвертированную dark-палитру PrimeVue).
- **Токены и компоненты** перечислены явно (`--mg-*` ⇄ `$…`/`--p-…`, `<Button variant=…>`,
  `<Tag severity=…>` и т.д.). Никаких новых цветов/радиусов/теней вне токенов.

## 🅓 Reuse-first (строго)
Сначала ищи существующий компонент/паттерн в `components/`, `ui_kits/crm/` и в
`front/src/components/**`. **Новый компонент — только с письменным обоснованием** в ТЗ
(почему существующие не подходят). По умолчанию — композиция готовых.
