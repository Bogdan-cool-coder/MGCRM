# ТЗ: Карточка контакта и компании (редизайн v2)

> Эталонный макет: **`redesign/entity-card.html`** (интерактивный, обе темы, переключатель Контакт/Компания, Tweaks). Этот документ описывает, как перенести его в продакшн `front/`. Все значения — на токенах дизайн-системы; в `.vue`/`.scss` пиши переменные репо (`$primary-900`, `$space-*`, `$radius-*`, `--p-*`), не литералы.

## Где в коде
- **Контакт:** `front/src/pages/ContactPage/index.vue` (+ `components/`)
- **Компания:** `front/src/pages/CompanyPage/index.vue` (+ `components/`)
- **Общие части:** `front/src/components/crm/entity/` — `EntityInfoHeader.vue`, `EntityKpiStrip.vue`, `InfoPanel.vue`, `EntityActivitiesTab.vue` (композер), `EntityLogTab.vue` (слит в Активность), таб-компоненты.
- Иконки — **PrimeIcons** (`pi pi-*`). Шрифт — Inter, база 14px.

## Общий каркас (сверху вниз)
1. **Тёмный navy-хедер** (бренд-инвариант `#172747`)
2. **Строка KPI-чипов** (белый фон)
3. **Таб-бар**
4. **Тело таба** (фон страницы `$surface-100` / `--c-page`, паддинг `16px 20px`)

Карточка — белая, бордер 1px `#E3E4E6`, радиус `$radius-lg` (8px), тень `$shadow-md`. Контентная зона — одна колонка на всю ширину (правого рейла НЕТ — его данные ушли в хедер).

## 1. Navy-хедер `EntityInfoHeader`
Фон `#172747` (`$brand-header-bg`), паддинг `16px 20px`. Внутри — flex-строка: **аватар → инфо-колонка (flex:1) → колонка управления**.

- **Аватар:** круг **56×56**, фон `rgba(255,255,255,0.14)`, инициалы (2 буквы), 600, ~20px, белые.
- **Инфо-колонка:**
  - **Строка 1 (имя):** `<h1>` **22px / 600 / #fff**, letter-spacing −0.01em. Справа от имени в той же строке (gap 10px, wrap):
    - **бейдж статуса клиента** (`Tag` success: «Активный клиент», иконка `pi-verified` 10px) — у компании; у контакта «Активный».
    - **бейдж категории** (компания) — `Tag` по `category_code`: L=danger, M=warning, S1=success, S2=info. 11px.
    - **Вовлечённость** — отдельная подпись `● Высокий` (точка 7×7 цвет `#A7EFAA`, текст 12px/600 `#bff5c0`). ⚠ Это НЕ статус клиента (см. ниже).
  - **Строка 2 (мета):** flex, gap 18px, wrap, 12px. Каждый пункт: `label:` (`rgba(255,255,255,0.4)`) + `value` (`rgba(255,255,255,0.8)/500`). Поля: **Автор · (Компания | Ответственный) · Источник · Создан · Изменён**. Подзаголовок (должность контакта) — первым, 13px `rgba(255,255,255,0.6)`. У компании подзаголовка НЕТ (ТОО/БИН убраны).
  - **Строка 3 (теги):** иконка `pi-tag` 10px + чипы тегов (11px/600, фон `rgba(255,255,255,0.12)`, текст `rgba(255,255,255,0.85)`, радиус `$radius-sm`).
- **Колонка управления** (справа, flex-shrink:0): только два значка в ряд (gap 6px) — `pi-arrow-left` (назад) и `pi-ellipsis-v` (меню). Кнопки 28-влкл, прозрачные, border `rgba(255,255,255,0.22)`, иконка 14px, hover `rgba(255,255,255,0.12)`.
  > Кнопки Задача/Звонок/Письмо из хедера УБРАНЫ. Звонок/письмо — на иконках в «Каналах связи»; задача — в «Активности».

### Меню «⋮» (popover, ~218px, белый, радиус md, тень lg)
Пункты (13px, иконка 13px): **Добавить заметку** (`pi-comment`) · **Добавить связь** (`pi-link`) · **Скопировать ссылку** (`pi-copy`) · разделитель · **Удалить** (`pi-trash`, danger `#FF5A44`).

## 2. KPI-чипы `EntityKpiStrip`
Белый фон, паддинг `13px 20px`, flex-wrap, gap 8px. Каждый чип — пилюля (`radius 999px`, padding `6px 13px`, 13px): иконка 12px + `label:` (opacity .8) + **значение 700**. Тинт по акценту:
- info → `--mg-blue-100 / --mg-blue-700`; brand → `--mg-primary-100 / --mg-primary-900`; success → green-100/900; teal → `#d4f3e8 / #0A6E4E`; amber → orange-100/900.

**Контакт:** Сделки (info, `pi-briefcase`) · Сумма (brand, `pi-wallet`) · Задачи (amber, `pi-check-square`) · Компании (teal, `pi-building`) · **Послед. активность** (success, `pi-clock`).
**Компания:** Откр. сделки (info) · Сумма (brand) · Выиграно (success, `pi-trophy`) · Сотрудники (teal, `pi-users`) · Документы (amber, `pi-file`) · Послед. активность (success).

Деньги: `1 200 000 ₽` / `≈ 6,8 млн ₽`.

## 4. Обзор — стопка `InfoPanel`
Белая карточка-контейнер (бордер, радиус lg, тень card), внутри сворачиваемые панели (`InfoPanel`), разделённые бордером. Шапка панели (мин-высота 44–46px, паддинг `11px 16px`):
**иконка-плитка 26×26** (радиус sm, фон `--p-primary-100`, иконка 12px) → **заголовок 14px/600** → **счётчик** (сразу справа от заголовка: 11px/700, фон `--c-hover`, радиус 10px, паддинг `1px 8px`) → flex-спейсер → **действие «+ …»** (12px/600, цвет `$primary-light`, hover-фон плитки) → **шеврон** (`pi-chevron-down/right` 12px).

Панели и их «+»-действия:
- **Контакт:** Каналы связи (`pi-phone`, +Создать канал) · Компании (`pi-building`, +Добавить в компанию) · Связи (`pi-share-alt`, +Добавить связь) · Участвует в сделках (`pi-briefcase`, +Добавить в сделку) · Заметки (`pi-comment`, +Добавить заметку) · История событий (`pi-history`).
- **Компания:** Реквизиты (`pi-id-card`, +Добавить реквизит) · Сотрудники (`pi-users`, +Добавить сотрудника) · Сделки в работе (`pi-briefcase`, +Создать сделку) · Холдинг (`pi-sitemap`, +Привязать компанию) · История событий (`pi-history`).

### Содержимое панелей
- **Каналы связи:** строки `[круглая иконка-действие 30×30] значение … подпись/действие [pi-copy]`. Иконка-кружок (фон плитки, иконка `pi-phone/pi-envelope/pi-telegram`) на hover становится navy с белой иконкой, а подпись справа меняется на действие (**Позвонить / Написать / Открыть чат**). Это и есть привязка действий к каналам.
- **Компании / Связи / Сотрудники-в-обзоре:** строки с `pi-building`/`pi-user`, имя-ссылка (`$primary-900`/500), бейдж «Основная/Контактное лицо».
- **Участвует в сделках / Сделки в работе:** мини-таблица в 3 выровненные колонки **Сделка (по ширине) · Этап (чип) · Сумма (справа, 700, navy)**. Без заголовка таблицы.
- **Реквизиты (мульти-юрлицо):** стопка под-карточек, каждая = юрлицо. Шапка под-карточки: плитка `pi-id-card` 24×24 + название юрлица (13px/600) + бейдж «Основной» + иконки **Копировать реквизиты** (`pi-copy`) и **Редактировать** (`pi-pencil`). Поля в 2 колонки (label uppercase 10px + value 13px/500), длинные значения переносятся целиком, на hover у поля — `pi-copy`.
- **Холдинг:** дерево (иконка `pi-sitemap`, отступ дочерних, бейдж «текущая»).
- **История событий (в обзоре):** короткий список (иконка-кружок 26 + текст + время).

## 5. Вкладка «Активность» (`EntityActivitiesTab`)
Лента **читается снизу вверх**: самое свежее — внизу у композера; вверху пометка `↑ более ранние события выше`, ниже разделитель-дата.
- **Фильтр-чипы** сверху: `Все / События / Изменения` (пилюли, активная — `--mg-primary-100`).
- **Карточка-событие:** бордер, радиус md, паддинг `11px 13px`: строка `[иконка типа 13px] заголовок (13px/600) [Tag статуса]`; тело заметки 13px `$surface-700`; подпись `actor · время` 12px muted. Иконки типов: звонок `pi-phone`, встреча `pi-calendar`, заметка `pi-file-edit`, задача `pi-check-square`.
- **Изменение (лог-строка):** компактно, не карточка — `[кружок 22px иконка `pi-pencil`] **Кто** изменил этап `старое` → `новое` … время`.

### Композер (внизу, фон `--c-hover`, паддинг `12px 16px`)
Слева — **две кнопки-режима друг под другом** (110px, 8px×12px): **Заметка** (`pi-comment`) и **Задача** (`pi-check-square`); активная — заливка navy/белый текст (по умолчанию «Заметка»). Справа — поле в рамке (`radius md`, бордер, паддинг 8px, **мин-высота 78px** = высоте двух кнопок), кнопка **«Добавить»** (primary) **справа по вертикали по центру**, текст слева переносится не задевая кнопку.
- **Режим Заметка:** одна `textarea` (без своей рамки, прозрачная) во всю высоту слева + Добавить справа.
- **Режим Задача:** в той же рамке две строки — верхняя: селекторы **дата/время (`pi-calendar`, открывает календарь) → Ответственный (`pi-user`, дропдаун) → Тип задачи (`pi-tag`, дропдаун)**; нижняя: `:` + поле текста задачи (textarea, переносится). Добавить так же справа по центру. Размер поля идентичен режиму Заметка.

## 6. Вкладка «Сделки» (полная)
Белая карточка + `TabHead` («СДЕЛКИ» 12px/700 uppercase + кнопка **Создать сделку** primary `pi-plus`). Таблица: **Название** (ссылка) · **Этап** (`Tag` по этапу) · **Сумма** (справа, 700 navy) · **Ответственный** · **Создана**. Строки hover `--c-hover`, бордер-снизу, паддинг ячеек `10px 14px`/13px.

## 7. Вкладка «Сотрудники» (компания)
`TabHead` + **Добавить сотрудника** (`pi-user-plus`). Таблица: **ФИО** · **Должность** · **Статус** (`Tag` success «Работает» / secondary «Уволен») · **Осн.** (звезда `pi-star-fill` оранж `#FFB38A` / `pi-star` серая) · **⋮**.
- **Раскрытие строки:** клик по строке (шеврон `pi-chevron-right/down` слева в ячейке ФИО) разворачивает под строкой контактные данные (телефон/email/Telegram чипами, паддинг слева 48px). Клик **по имени** (ссылка, stopPropagation) — переход в карточку контакта.
- **Меню «⋮»:** Перейти в карточку (`pi-user`) · Сделать основным (`pi-star`) · Изменить статус (`pi-sync`) · разделитель · Отвязать контакт (`pi-user-minus`, danger).

## 8. Вкладка «Документы» (компания)
`TabHead` + **Создать документ**. Колонки (именно в этом порядке): **№** · **Статус** (`DocumentStatusTag`: Подписан=success, На согласовании=warning, Черновик=info) · **Вид** · **Дата** (`ДД.ММ.ГГГГ`, напр. `18.06.2026`) · действия (`pi-external-link`, `pi-file-pdf`).

## 9. Вкладка «Файлы» (контакт и компания)
`TabHead` + две кнопки: **Создать папку** (`pi-folder-plus`, secondary) и **Загрузить файл** (`pi-upload`, primary). Тело — две панели:
- **Слева (46%, бордер справа):** список папок. Строка: `pi-folder`/`pi-folder-open` (выбранная) 15px + название (13px, выбранная 600) + счётчик файлов (пилюля) + `pi-chevron-right`. Папки: **Папка менеджера сделки · Сканы договоров / ДС · Папка менеджера ОКС**.
- **Справа:** файлы выбранной папки. Строка: иконка по типу (`pi-file-pdf/word/excel/file` 16px) + `имя` (13px/500) + `размер · дата` (11px muted) + `pi-download` + `pi-ellipsis-v`.

## 10. Вкладка «Холдинг» (компания)
`TabHead` + **Привязать компанию**. Дерево холдинга.

## 12. «Активный клиент» ≠ «Вовлечённость» (важно)
- **Статус клиента** (`client_status`, `ClientStatusBadge`): договорные отношения — «Активный клиент» / «Отключён». Ставится вручную / по сделкам.
- **Вовлечённость** (`engagement_tier`, `EngagementChip`): насколько «живой» контакт сейчас — считается автоматически от `last_activity_at` (Высокий/Средний/Низкий). Можно быть активным клиентом с низкой вовлечённостью — это сигнал «пора связаться». Дубля нет.

## 13. Токены / общие правила
- **Радиусы:** input/tag/badge `$radius-sm` (4), button/card/panel `$radius-md` (6), dialog/большие карточки `$radius-lg` (8), аватар-плитка `$radius-xl` (12); круглые — аватар, count-пилюли.
- **Отступы:** гаттер тела `20–24px`, паддинг карточки/панели `16px`, ячейки таблиц `10px 14px`, шкала `$space-1…8`.
- **Тени:** карточка = бордер 1px + `$shadow-card`; hover-строки/карточки → `--c-hover` / `$shadow-md`; диалоги `$shadow-lg`. Без цветных теней.
- **Статусы** — мягкие тинт-пилюли (success/danger/warning/info). **Иконки только PrimeIcons, без эмодзи.** Тексты — RU, sentence-case, через i18n.
- **Обе темы обязательны** (navy-хедер и шапка карточки сделки — инвариантны). В макете тёмная палитра: карточка `#444547`, фон `#272829`, текст `#F9FAFB`, бордер `#616263`.
