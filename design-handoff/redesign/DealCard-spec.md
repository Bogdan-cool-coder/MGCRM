# Карточка сделки (DealPage 2.0) — ТЗ для реализации (Claude Code)

Пересборка карточки сделки: двухпанельный экран (слева данные + табы, справа лента
событий + открытые задачи + composer). Эталон — `redesign/deal-card.html`. Документ
описывает разметку, размеры, токены и поведение так, чтобы фронтендер (агент
`frontend-specialist`) собрал экран пиксель-в-пиксель.

- **Стек:** Vue 3 + PrimeVue 4. Страница — `front/src/pages/DealPage/index.vue`; компоненты —
  `DealInfoPanel`, `DealInfoHeader`, `DealInfoTabs`, `DealTabMain`, `DealTabDocuments`,
  `DealTabFinances`, `DealFeed`, `DealFeedItem`, `DealComposer`, `OpenTasksList`,
  `DealProductsGroup`, `DealContactsGroup`, `DealCompanyGroup`, `DealFieldGroup`, `MoveDealDialog`,
  `DealAddProductDialog`, `DealAddContactDialog`.
- **Шрифт:** `--mg-font-sans` (SF UI Display → Inter). Иконки — **PrimeIcons 7**.
- **Темы:** светлая и тёмная (класс `.surface` / `.surface.dark`).

- **Строка 2 (стадия):** flex wrap, gap 8.
  - **Тег стадии** — заливка цветом этапа, 12px/500, белый, без шеврона. Клик → диалог смены
    стадии (§8.1).
  - **Категория компании** — бейдж (11px/700, белый, `background:--mg-red-600`, radius sm). Текст = L/M/S.
  - **Health-чип** (§2.1.1).

- **Строка 3 (теги):** flex wrap чипы — `rgba(255,255,255,.12)` фон, белый текст, pill, иконка
  `pi-tag` 8px. Показывать, если у сделки есть теги. *(Теги редактируются через ⋮ → «Изменить теги».)*
- **Прогресс-бар этапов:** flex, gap 2, height 6, radius sm, overflow hidden. Сегмент на каждый
  видимый этап: пройденные/текущий — цвет этапа, будущие — `rgba(255,255,255,.2)`, won —

#### 2.1.1 Health-чип (следующая задача)
pill (radius sm, 12px, иконка 9px). Состояния: **нет задачи** → `pi-clock`, фон `rgba(234,179,8,.12)`,

#### 2.1.2 Меню «⋮»
Открывается **справа от кнопки** (поповер). `min-width:210; padding:5; radius md; shadow lg`.
Пункты (13px/500, иконка 15px, hover `--c-hover`, разделители — линия): **Копировать ссылку**
(вверху) · — · Переименовать · Изменить теги · — · Свернуть все группы · Развернуть все группы ·
Настроить поля · — · **Удалить сделку** (красный).

## 3. Таб «Основное» (`DealTabMain`)

### 3.1 Быстрые поля
`padding:12px 0 8px; border-bottom`. Каждое поле — grid `110px 1fr`, min-height 32, `padding:5px 16px`,
лейбл 12px `--c-muted`:
- **Ответственный** — аватар-кружок 22px (нэйви, инициалы) + имя (13px). Клик по полю открывает
  **поповер выбора ответственного с поисковой строкой** (§9). Карандаша нет.
- **Компания** — ссылка (`--mg-primary-900`, 500).
- **Договор план** — поле даты (§9, без дефиса в подписи).
- **Оплата план** — поле даты (§9).

### 3.2 Группы (`DealFieldGroup`)
Заголовок-полоска: `padding:8px 16px; min-height:36`, **клик по всей полоске** сворачивает/
разворачивает (шеврона/треугольника нет). У **всех** групп иконка — в виде плитки 24×24
(`background:--mg-primary-100; color:--mg-primary-900; radius sm`). Accent-группы: фон полоски
`--c-sub`, заголовок 12px/600 UPPERCASE `--c-text`. Состав: иконка-плитка · название · [бейдж-счётчик]
· [сумма] · спейсер · [действие справа].

**Продукты** (accent, открыта): сумма «4 500 000 ₸ · 12 мес» сразу после заголовка; справа действие
«+ Добавить» (открывает §8.2). Счётчик кол-ва **не показывать**. Внутри:
- **Лицензия** — сегментированный переключатель (НЕ тумблер): `[Подписка | Вечная]` — контейнер
  `background:--mg-gray-100; radius 7; padding 3`, активный сегмент — `background:--c-card;
  shadow sm; color:--mg-primary-900`. Справа в той же строке — **Скидка [ ]%** (числовое поле 70px + «%»).
- Строки продуктов: название (13px) + «{период} × {цена}» (11px `--c-muted`); справа сумма (13px/600).
  Период = 1/3/6/12/24 мес или «разово». Строки «Итого» внизу нет (дублирует сумму в заголовке).

**Контакты** (accent, открыта, счётчик): на каждый контакт — имя-ссылка + бейдж «Основной»
(`--mg-blue-700` на `--mg-blue-100`); справа **⋮** → меню **Редактировать / Отвязать**
(Отвязать с иконкой `pi-times-circle`). В режиме «Редактировать» строка превращается в **полную
форму**: поля **Имя** · **Должность** · редактируемые **каналы связи** (значок канала + поле
значения + ✕ удалить + «+ Добавить канал») · тумблер **«Основной контакт»** (вкл/выкл) +
Отмена/Сохранить. Должность (12px `--c-muted`), ниже — чипы каналов (иконка
канала + значение). Кнопка **«+ Добавить канал»** (dashed) → поповер: тип канала (выпадающий
список) + значение + «Добавить». Внизу группы — «+ Добавить контакт» → диалог §8.3.

**Компания** (accent, свёрнута): строки grid `1fr auto` в порядке: **Телефон · Email · Сайт ·
Адрес · Тип компании**. (БИН и Категория не показывать — категория в шапке.)

**Дополнительные поля** (accent, свёрнута): Источник лида · **Используют** (бывш. «Конкурент»).

## 7. Правая панель (`DealFeed` + `OpenTasksList` + `DealComposer`)
Фон `--c-feed`.

### 7.1 Topbar — виджеты ключевых действий + поиск
`padding:8px 14px; border-bottom; background:--c-card`. Слева — **нейтральные** pill-чипы
(`background:--c-hover; border:1px --c-border; color:--c-text2`; иконка `--c-muted`, значение 700):
Презентация:10.06 · КП:12.06 · Касание:18.06 · Событие:15.06. **Клик по чипу = быстрый поиск:**
прокручивает ленту к соответствующему выполненному событию и подсвечивает его (рамка
`--mg-primary-900` + кольцо `--mg-primary-100`, ~2 c). **Чипы НЕ создают новых событий.** Справа —
иконка `pi-search` → поповер поиска (строка + чипы типов: Все · Смена стадии · Изменения полей ·
Примечания · Задачи · Звонки · Встречи + «Сбросить» справа от заголовка «Тип»).

### 7.2 Лента (`DealFeed`, `DealFeedItem`) — снизу вверх
`overflow-y:auto`, `justify-content:flex-end` (новое внизу, у composer'а). Группы по датам:
разделитель «— ДАТА —» (клик по полоске сворачивает группу, без иконки). Внутри группы — вертикальная
линия-таймлайн слева.
- **Системные события** (создание, смена стадии, изменение полей): **серый текст без карточки** —
  кружок-иконка 22px `--c-hover`, «{Автор} {действие} <s>старое</s> → новое», время справа.
- **Заметки и задачи** — карточка (`--c-card`, border, radius md): кружок-иконка 26px (цвет dot:
  primary/green/red/surface). Заголовок (note — 400, иначе 600); справа бейдж статуса для задач
  (Выполнено — green / Открыта — orange) + время. Опц. тело (12px `--c-text2`), строка due/
  ответственный (иконки). Внизу — автор (11px `--c-muted`). Выполненные — `opacity:.72` + зачёркнутый
  заголовок. У карточек ключевых действий — `data-ka="<type>"` для быстрого поиска (§7.1).

> *(as-built 2026-06-27: бейдж статуса задачи в ленте должен использовать **реальный 4-state enum**
> `new` / `in_progress` / `done` / `rejected` (см. `ActivityStatus` + `payload.status`), а **НЕ
> бинарный done/open**. Бинарный бейдж в эталоне `deal-card.html` (~L530–531, `{done ? 'Выполнено'
> : 'Открыта'}`) **устарел** — .html не правим, источник истины здесь.)*

### 7.3 Открытые задачи (`OpenTasksList`)
Полоска «Открытые задачи: N» (клик сворачивает список; **по умолчанию открыт**). Каждая задача —
карточка (`--c-sub`, border): слева **значок типа задачи** (pi-phone/pi-check-square), затем
заголовок + строка (дата · ответственный). Дата и ответственный — **кликабельны** (§9: дата →
календарь, ответственный → список с поиском). **Клик по задаче раскрывает** действия: поле
«Добавить результат» (textarea) + **Выполнить** (зелёная) + **Удалить** (красный текст).

### 7.4 Composer (`DealComposer`)
`border-top; padding:12px 16px; background:--c-card; display:flex; gap:10; align-items:center`.
Слева — две кнопки-режима друг под другом (108px): **Заметка** / **Задача** (активная — заливка
`--mg-primary-900`, белая). Справа — поле в рамке (min-height 74; кнопка **Добавить** по центру
по вертикали — на уровне между кнопками Заметка/Задача):
- **Заметка:** textarea «Введите текст заметки…» + «Добавить».
- **Задача:** верхняя строка из 3 полей в один ряд (§9): **дата+время** · **ответственный** (поиск)
  · **тип задачи** (Задача/Звонок/Встреча/Follow-up); ниже «: {textarea текст задачи}»; справа «Добавить».

## 8. Диалоги
Модалка: overlay `rgba(0,0,0,.4)`, центр; карточка `radius lg; shadow lg`; шапка (заголовок +
`pi-times`), тело (`padding:18; max-height:70vh; overflow-y:auto`), футер (Отмена + основное действие).
Обязательные поля — красная звёздочка (`--mg-red-600`).

### 8.1 Смена стадии (`MoveDealDialog`)
Текущая стадия (точка цвета + имя) + «Новая стадия» — список стадий (точка цвета + имя, выбранная
залита `--mg-primary-100` с `pi-check`). Футер: Отмена / Переместить.

### 8.2 Добавить продукт (`DealAddProductDialog`)
Порядок строго: **Период** (select) · **Валюта** (select) · **Продукт** (выпадающий список с
поиском) · **Сумма (авто)** — рассчитывается по периоду и цене продукта (read-only плашка `--c-sub`).
Полей «Тарифный план» и «Цена/ед. (снимок)» **нет**.

### 8.3 Добавить контакт (`DealAddContactDialog`)
Поле «Контакт» — поиск по базе; список раскрывается **только по клику/фокусу** (не открыт по
умолчанию). Если контакта нет — кнопка **«Создать новый контакт»** → форма: **Имя*** · **Телефон*** ·
Email · **Должность*** (выпадающий список с поиском) · чекбокс «Основной контакт». (Имя, телефон,
должность — обязательные.)

## 9. Поля даты и выпадающие списки (единый паттерн)

**Поле даты** (`DateField`): редактируемый `<input>` + иконка `pi-calendar`. Клик по всему полю →
поле становится активным для ввода **и** открывается календарь (ниже/выше по месту). Ввод —
**авто-формат ДД.ММ.ГГГГ** (точки ставятся сами после 2 и 4 цифр); календарь синхронно подсвечивает
введённый день. Календарь — мини-грид (Пн–Вс + дни, выбранный день залит `--mg-primary-900`).
Поля **Договор/Оплата (план и факт)** — **только дата, без времени**. Дублирующей строки ручного
ввода внутри календаря нет — единственный источник ввода это само поле.

**Выпадающий список с поиском** (`SearchPicker` / поповер): триггер (значение + шеврон) → поповер
с поисковой строкой (`pi-search`, «Начните вводить…») + список опций (выбранная залита
`--mg-primary-100` с `pi-check`). Применяется: ответственный (везде), тип задачи, должность, шаблон
документа, валюта, тип канала, контакт. В composer'е задачи дата+время поддерживает ввод времени
отдельным полем.

## 11. Свежие правки (финальная итерация — учесть обязательно)

**Лента событий:**
- В ленте — **только свершившиеся действия** (выполненные задачи, заметки, системные изменения).
  Открытые задачи в ленте НЕ показываются — они только в списке «Открытые задачи» под лентой;
  задача попадает в ленту в момент выполнения.
- **Системное сообщение — одной строкой** в порядке: **Автор · дата · время · действие ·
  результат**. Старое значение зачёркнуто тонкой линией, новое — обычным текстом. Пример:
  «Богдан Меркулов, 18 июн, 15:40 изменил бюджет ~~3 800 000 ₸~~ → 4 500 000 ₸». Аналогично для
  смены статуса, ответственного, телефона контакта и любых других полей.
- **Значок типа задачи в карточках ленты подсвечивается цветом** на светлой плашке-кружке, рамка
  карточки тонко в цвет типа: звонок — синий `#2A6FDB` (`pi-phone`), встреча — зелёный `#1F8A5B`
  (`pi-calendar`), КП/предложение — жёлтый `#E8A317` (`pi-file-check`/`pi-file-edit`), договор —
  красный `#D14343` (`pi-file`), задача — нэйви. Заметки/системка — нейтральные.
- В карточке заметки/задачи **дата · время · ответственный — в одну строку** (📅 · 🕐 · 👤).
- Лента скроллится снизу вверх (`margin-top:auto`, не `justify-content:flex-end`), скроллбар скрыт.

**Открытые задачи (`OpenTaskRow`):**
- Поля в строке в порядке **Тип · Дата · Ответственный** (значка типа слева от заголовка нет —
  тип показан цветным чипом в строке мета). Все три кликабельны (тип/ответственный — списки,
  дата — календарь с авто-форматом ДД.ММ.ГГГГ, без времени для план/факт).
- **Заголовок задачи** — редактируемое многострочное поле (перенос по краю): **один клик** по
  карточке раскрывает её; свернуть можно **только кликом вне карточки**; внутри раскрытой карточки
  любые клики не сворачивают; **двойной клик** по тексту включает редактирование (blur — выключает).
- **«Выполнить»** — в правом верхнем углу, видна всегда; в раскрытом виде клик подсвечивает поле
  «Опишите результат…» как обязательное (красная рамка), после заполнения повторный клик завершает.
- **Удаление** — маленький серый ✕ справа от «Выполнить»; удаляется по **трём кликам** (после
  первого краснеет/увеличивается, в подсказке — сколько кликов осталось).

**Поля даты — единый компонент:** ввод вручную в самом поле (авто-формат ДД.ММ.ГГГГ, точки сами) +
календарь открывается по клику на поле; план/факт — без времени; дублирующей строки внутри
календаря нет.
