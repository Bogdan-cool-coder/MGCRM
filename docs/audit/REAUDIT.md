## Что закрыли в Wave 4 (главное)

**Blockers:**
- Удаление задачи в Open-tasks теперь доходит до API (`feed.deleteActivity`) — не воскресает после reload (карточки сделки + компании/контакта).
- IDOR вложений документов: `download/destroy` проверяют принадлежность вложения документу (подтверждено живьём: чужое вложение → 404).

**Регрессии от наших же фиксов (откат):**
- `/api/admin/*` справочники: **read открыт авторизованным** (менеджерам нужны фильтры/лейблы), **write — только админ** (`can:admin-write`). Раньше Wave-1 перекрыл и чтение → пустые дропдауны. Замок e2e обновлён под этот контракт.
- `lawyer`-scope унифицирован через единый `VisibilityResolver` (list/KPI/item совпадают).
- `CompanyChannelsBlock` импортирован (блок каналов рендерится).

**Majors:** кластер contracts-templates (фильтры kind/search/var, CRUD лицензиаров + утечка `tax_id`/реквизитов закрыта политикой, per-currency банк-счёт в генерацию, автор версии), `is_closed` больше не mass-assignable (обход стейт-машины), админ-UI справочника вопросов о встрече, онбординг (quiz-review shape + серверный soft-gate), видимость воронок в борде, дефолтный пайплайн дашборда, salespulse vacation skip-upgrade.

## Что осталось (осознанно)

1. **Catalog FX — ежедневный авто-refresh курсов.** Конвертация работает на засеянных курсах, но плановое обновление мертво — **нужен секрет `EXCHANGE_RATE_API_KEY`** (+ `EXCHANGE_RATE_API_URL`) в `src/.env`. Это действие на твоей стороне (внешний провайдер), не код.
2. **Продуктовые/архитектурные решения** (см. [REMAINING.md](REMAINING.md)): дроп колонки `users.role` (после переноса всех записей роли на API), invite/email-флоу + password-reset, Department CRUD/орг-дерево (спринт M1), TTL Sanctum-токенов, `reschedule`-эндпоинт (спека ↔ FE расходятся), удаление скелет-таблиц.
3. **Чистые CONVENTION-нити** (например, доменные Policy читают синкнутый `users.role`-зеркало вместо `can()`): после IAM-1 зеркало авторитетно-синкается, поведение корректно — это косметика, не баг; оставлено намеренно.

## ⚠️ Обязательно при деплое (IAM-1)
Смена guard на `sanctum` осиротит старые web-guard роли на проде. Деплой ДОЛЖЕН прогнать: `php artisan db:seed --class=RolePermissionSeeder` → ре-синк ролей всех юзеров (`User::each(fn($u)=>$u->syncRoles([$u->role->value]))`) → `php artisan permission:cache-reset`. Иначе все получат 403. Выполнить через `deploy-engineer`.
